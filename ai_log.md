- 내가 입력한 프롬프트:  (지금까지 만든 코드를 복사하고 붙여넣었었습니다.) 현재 코드를 직접 테스트해 보고 있는데, 사용자가 입력한 조건과 맞지 않아 계산된 최종 점수가 0점인 음악들까지 불필요하게 추천 결과 TOP 3에 섞여 나오는 현상이 있어. 이 오류를 해결할 수 있는 코드 로직을 알려주고, 네가 이 전체 코드를 한 번 디버깅 해서 그 외의 에러가 있다면 그 에러도 완벽하게 차단할 수 있는 디버깅 수정 코드를 알려줘.


- Gemini가 제안한 내용 요약: 사용자가 입력한 장르와 목적이 맞지 않아 최종 점수가 0점인 음악이 순위에 노출되는 현상을 발견함. 최종 추천 점수 계산 부근에 if score > 0: 조건문을 추가하여, 점수가 0점을 초과할 때만 결과 리스트(scored_list)에 추가(append)하도록 필터링하는 로직을 제안함.
- 내가 반영한 부분:
- 내가 수정하거나 사용하지 않은 부분:입력 데이터 예외 차단 및 형식 재입력을 유도하기 위해 다소 길고 복잡하게 제시했던 무한 반복 루프(while True)와 try-except 구문은 오리지널 코드의 간결함과 직관성을 저해할 수 있다고 판단하여 채택하지 않음. 대신 핵심적인 데이터 필터링 논리(if score > 0:)만 선택하여 내 코드에 직접 구현함.
- 반영 위치: main.py의 calculate_scores 함수 리스트 추가부

## AI 활용 성찰

Gemini가 도움이 된 점: 음악 데이터베이스의 데이터 균형을 맞추는 데 기여할 만한 대중적인 노래 목록을 제안해 주어 프로젝트를 풍부하게 구상할 수 있었습니다.또한 2차원 리스트 정렬 과정에서 lambda 식을 활용해 점수 기준 내림차순 정렬을 실행하는 간결한 코드를 제시해 주어 백엔드 개발자로서의 알고리즘 활용 역량을 쌓는 데 큰 도움을 얻었습니다. 무엇보다 사용자가 입력한 선호 장르와 목적에 일치하지 않아 추천 점수가 0점이 된 음악이 추천 순위에 노출되던 고질적인 논리 오류를 if score > 0:이라는 직관적이고 군더더기 없는 비교 필터링 조건문 제안을 통해 신속하게 해결할 수 있었습니다.

Gemini의 제안을 그대로 쓰지 않고 내가 판단하거나 수정한 점:
AI가 알려주는 코드를 무비판적으로 복사하여 붙여넣지 않고, 내 주체적인 판단에 맞추어 변형 및 제어했습니다. 최초 음악 목록 데이터를 구성할 때 제안받은 점수를 그대로 쓰지 않고 내 주관적인 곡 해석 기준에 맞게 속성 점수를 조율했으며 좋아하는 곡을 직접 골라 커스터마이징했습니다. 또한 Gemini는 입력을 오류 방지를 위해 다중 반복 루프와 예외 복구 구문(try-except)을 대거 포함한 이해하기 어렵고 무거운 코드를 제안했지만, 본래 코드의 구조적 직관성을 잃지 않기 위해 복잡한 입력 루프 구문은 전량 생략하고 데이터의 정합성을 걸러내는 단순 명료한 비교 필터링 로직(if score > 0:)만 발췌하여 주체적으로 오류를 해결했습니다.

다음에 AI를 사용할 때 주의해야 할 점: AI는 개발 생산성 향상과 논리 구조 수립에 훌륭한 역할을 해주지만, 때로는 가독성을 해치거나 현재 내가 배운  범위를 과도하게 초과하는 지나치게 복잡한 조건문 패턴이나 제어 구조를 제시할 수 있음을 알게되었습니다. 따라서 다음에도 AI의 조언을 구할 때는 제공받은 처리 과정을 내 힘으로 한 줄 한 줄 모두 설명할 수 있는지 확인하며 ai를 효율적으로 사용해 주체적인 소프트웨어 설계 습관을 지녀야겠다고 다짐했습니다.
