# 프로젝트 계획서

## 1. 프로젝트 주제
 
내가 만들 프로그램의 주제: 사용자 맞춤형 음악 추천 알고리즘 시스템

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명: 사용자가 좋아하는 노래 장르와 노래를 듣는 목적, 그리고 장르와 목적 둘 중 무엇을 더 중요하게 여기는지를 고려하여 사용자에게 맞는 노래를 추천해주고자 한다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 사용자의 닉네임
- 선호하는 음악 장르 2차원 리스트로 각각의 음악 정보(제목, 가수(그룹) 이름, 음악 장르 등등) 리스트로 구성된 음악 데이터베이스 사용
- 사용자가 음악을 들으려는 목적 & 장르와 목적의 각각의 중요도

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 사용자가 입력한 선호 장르와 음악 데이터베이스의 장르를 비교하여 일치하는 곡에 기본 추천 점수를 부여한다.
- 사용자의 목적에 따라 음악의 목적 점수를 추출하고, 중요도를 가중치로 장르 점수와 목적 점수에 곱해 최종 추천 점수를 계산한다.
- 계산된 최종 추천 점수를 기준으로 전체 음악 목록을 내림차순으로 정렬한다.

### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 사용자의 입력 정보 확인 메시지
- 최종 추천 점수가 높은 순서대로 정렬된 음악 목록 TOP 3

## 4. 2차원 리스트 설계

내 프로그램에서 사용할 2차원 리스트의 이름: music_db

예) foods, movies, characters, questions, records

```python
#music_db = [
#    ['노래 제목 1', '가수(그룹) 명', '음악 장르', 집중할 때 듣기 좋은지에 대한 점수(정수형), 분위기를 띄울 때 듣기 좋은지에 대한 점수(정수형)],
#    ['노래 제목 2', '가수(그룹) 명', '음악 장르', 집중할 때 듣기 좋은지에 대한 점수(정수형), 분위기를 띄울 때 듣기 좋은지에 대한 점수(정수형)],
#]
```

### 한 행의 의미
등록되어 있는 음악 곡 1개에 대한 고유한 속성 및 점수 데이터 정보 세트

### 각 열의 의미

| 열 번호 | 의미 | 예시 |
|---:|---|---|
| 0번 열 | 노래 제목 | "소문의 낙원" |
| 1번 열 | 가수(그룹) 명 | "AKMU(악뮤)" |
| 2번 열 | 음악 장르 | "포크" |
| 3번 열 | 집중할 때 듣기 좋은지에 대한 점수 | 90 |
| 4번 열 | 분위기를 띄울 때 듣기 좋은지에 대한 점수 | 30 |

## 5. 함수 설계

| 함수 이름 | 역할 | 입력값 | 반환값 또는 출력 |
| calculate_scores | 데이터베이스를 순회하며 사용자 장르 일치 여부와 목적이 무엇인지에 따라 장르점수와 목적 점수를 부여하고 장르와 목적의 중요도를 기반으로 가중치를 곱해 최종 점수를 계산 및 리스트화 | db (2차원 리스트), user_genre (선호장르), user_purpose (목적), genre_weight (장르 중요도 비율), purpose_weight (목적 중요도 비율) | scored_list ([제목, 가수, 최종 점수]가 한 행인 2차원 리스트) |
| sort_and_print | 입력받은 추천 데이터 리스트를 최종 점수 기준으로 정렬한 뒤 TOP 3 곡을 정돈된 규격에 맞춰 출력 | scored_list ([제목, 가수, 최종 점수]가 한 행인 2차원 리스트) | 최종 사용자 맟춤 음악 TOP 3 순서대로 출력 |
| main | 사용자로부터 입력을 차례로 받고 연산 및 출력 함수들을 호출하는 주 흐름 제어 | 사용자의 입력(닉네임, 선호 장르, 목적, 중요도)을 수신함 | 입력받은 값 확인하는 문구 출력, 계산 및 출력 함수 호출 |

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획: 2차원 리스트 내부의 음악 장르와 사용자가 입력한 선호 장르가 일치하는지 확인하고 점수를 부여한다.
사용자가 입력한 목적이 무엇인지에 따라 study_score(집중하기)와 sports_score(분위기띄우기) 중 목적 점수가 될 점수를 정한다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획: 2차원 리스트에 있는 모든 음악의 데이터를 한 행씩 차례대로 끝까지 접근하기 위해 조건문을 사용한다.
정렬 완료된 음악들 중 사용자에게 상위 순위인 TOP 3 결과만 출력하기 위해 조건문을 사용한다.
